## Platform — 2025-04-01

### Added
- **Billing system** — monthly usage-based billing via Paystack reusable authorisations. Charges run automatically at 02:00 UTC each day for subscriptions due.
- **Transactional email** — billing receipts, card-saved confirmations, tier-switch notifications, and welcome emails are now sent via Resend.
- **Rate limiting** — sliding-window rate limits applied to all API endpoints (file processing, payments, RPC routes). Limits are communicated via standard `X-RateLimit-*` headers.

### Changed
- `POST /api/file-processing/extract` now returns HTTP `202 Accepted` immediately and processes documents asynchronously. Poll `GET /api/file-processing/status/:jobId` for completion.

## SDK — v0.2.0-beta.3 (2026-04-04)

Beta release. See the [release notes](/docs/releases/v0.2.0) for full code examples.

### `@nexa-ed/sdk`
- **New:** `nexa.email` module — `create`, `bulkCreate`, `getJobStatus`, `list`
- New types: `EmailCreateOptions`, `EmailCreateResult`, `EmailBulkCreateOptions`, `EmailBulkCreateResult`, `EmailJobStatus`, `StudentEmailAccount`

- **18 components** total — email provisioning (`StudentEmailAccountManager`, `EmailBulkProvisioningPanel`), payments (`NexaPaymentWidget`, `EnrollmentPaymentFlow`, `PaymentStatusDashboard`, `PaymentConfigPanel`), document/results (`UploadZone`, `StudentRecordsTable`, `ResultsViewerSheet`, `AnalysisJobPanel`, `PipelineTimeline`)
- **15+ hooks** — `useUploadFile`, `useFileProgress`, `useGetUserDocuments`, `useGetFileStudentRecords`, `useGetPageRecords`, `useGetAnalysisJob`, `useGetFileStats`, `useRefinePageWithLLM`, `useUpdatePageRecord`, `useDeletePageRecord`, `useColumnSort`, `useGridNavigation`

### `@nexa-ed/next`
- Single `createRouteHandler(nexa)` wires all routes: file uploads, SSE progress, Paystack webhooks, payment forwarding, oRPC queries

### `@nexa-ed/convex`
- `nexaStudentEmailSchema` schema fragment
- `makePaymentWebhookHandler` factory

### `@nexa-ed/node`
- Express, Fastify, and Hono adapters (`createExpressAdapter`, `createFastifyPlugin`, `createHonoAdapter`)
- Standalone `verifyPaystackWebhook` helper

### `create-nexaed-app`
- Interactive CLI — prompts for Convex URL, Clerk keys, email tier, and domain
- Scaffolds Next.js + Convex app with all packages pre-wired

## Platform — 2025-03-17

### Added
- **oRPC data API** — typed `POST /api/rpc/[...orpc]` endpoints for querying student records, analysis jobs, and document chunks. Authenticated via Clerk JWT.
- **Auth hardening** — FastAPI now accepts Clerk JWTs (RS256, JWKS cached 1 hour) as well as internal shared-secret headers. Worker routes are secured via `x-internal-secret`.

---

## Platform — 2025-03-10

### Added
- **Document AI pipeline** — async pipeline: upload → Trigger.dev → FastAPI OCR → Mastra agents → NeonDB. Supports PDF and image uploads.
- **Student records schema** — `student_records`, `subject_categories`, `results_analysis_jobs` tables in NeonDB via Drizzle ORM.
- **Clerk → NeonDB bridge** — `POST /internal/provision-user` on the worker provisions Clerk users into NeonDB on first document upload.

## Platform — 2025-02-15

### Added
- Multi-tenant SaaS platform launch — tenant registration, provisioning, and API key management.
- Paystack payment gateway integration — fee collection, webhook verification, subscription management.
- Student email provisioning — Tier 1 (Nexa subdomain), Tier 2 (custom domain via Stalwart), Tier 3 (Google Workspace managed).
