---

`create-nexaed-app` scaffolds a complete Next.js school app with routing, auth, a results dashboard, and payment support already wired up. It's the fastest way to go from zero to a working integration.

<Callout type="info">
  Prefer a visual interface? **[Try the Stack Builder →](/builder)** — click to select your auth provider, features, email tier, and package manager, then copy the generated command.
</Callout>

---

## Commands

```bash
npx create-nexaed-app                      # scaffold a new school app (default)
npx create-nexaed-app init my-school       # explicit init
npx create-nexaed-app doctor               # validate an existing app
npx create-nexaed-app upgrade              # upgrade @nexa-ed/* deps to @latest
npx create-nexaed-app telemetry on|off     # manage telemetry preference
npx create-nexaed-app help                 # print usage
```

---

## Run the scaffolder

```bash
# npx (no install required)
npx create-nexaed-app my-school

# pnpm
pnpm create nexaed-app my-school
```

---

## Interactive prompts

The CLI asks six questions:

| Prompt | Options |
|--------|---------|
| School name | Used as directory name and `package.json#name` |
| Auth provider | WorkOS · Clerk · NextAuth · None |
| Features | File processing, Payments, Convex integration, Student email provisioning |
| UI library | None (plain Tailwind) · shadcn/ui |
| Initialize git? | Yes (default) · No |
| Nexa API key | Optional — can be added to `.env.local` later |

---

## Non-interactive / CI mode

All prompts can be bypassed with flags. Use this in CI pipelines, Dockerfiles, or shell scripts:

```bash
npx create-nexaed-app my-school \
  --auth clerk \
  --features fileProcessing,payments \
  --ui shadcn \
  --email-tier tier-1-nexa \
  --api-key nxa_live_... \
  --pm pnpm
```

| Flag | Description |
|------|-------------|
| `--auth` | `workos` · `clerk` · `nextauth` · `none` |
| `--features` | Comma-separated: `fileProcessing`, `payments`, `convex`, `emailProvisioning` |
| `--ui` | `none` (default) · `shadcn` |
| `--email-tier` | `tier-1-nexa` · `tier-2-stalwart` · `tier-3-google` |
| `--email-domain` | Your school domain (required for tier 2/3) |
| `--api-key` | Pre-fill the Nexa API key in `.env.local` |
| `--pm` | `pnpm` · `npm` · `yarn` · `bun` |
| `--no-install` | Skip dependency installation |
| `--git` | Initialize a git repository and create an initial commit |
| `--no-git` | Skip git initialization |

When stdin is not a TTY, the non-interactive fast path activates automatically — no flags needed in most CI environments.

## `doctor` — validate an existing app

Run `doctor` inside an existing Nexa-powered school app to check for common setup issues:

```bash
cd my-school
npx create-nexaed-app doctor
```

Runs 5 checks:

| Check | What it verifies |
|-------|-----------------|
| **Env vars** | `NEXA_API_KEY` and `NEXA_WEBHOOK_SECRET` present in `.env.local` |
| **Packages** | Required `@nexa-ed/*` packages listed in `package.json` |
| **Route** | Catch-all `app/api/nexa/[...nexaed]/route.ts` exists |
| **SDK version** | Your installed version vs latest on npm |
| **API key** | Live request to the Nexa platform confirms the key is valid |

---

## `upgrade` — update SDK packages

Updates all `@nexa-ed/*` dependencies in the current project to `@latest`:

```bash
cd my-school
npx create-nexaed-app upgrade
```

The package manager is detected automatically from your lockfile.

## What gets generated

```
my-school/
├── app/
│   ├── api/nexa/[...nexaed]/
│   │   └── route.ts        ← single catch-all route handler
│   ├── dashboard/
│   │   └── page.tsx        ← main dashboard
│   ├── layout.tsx
│   ├── page.tsx            ← landing page (Nexa brand colours)
│   ├── providers.tsx
│   └── globals.css
├── lib/
│   ├── nexa.ts             ← SDK instance (pre-wired to your auth provider)
│   └── utils.ts            ← cn() helper  [shadcn only]
├── middleware.ts            ← auth route protection  [clerk / nextauth only]
├── components.json         ← shadcn/ui config  [shadcn only]
├── tailwind.config.ts
├── .env.example            ← all required vars with inline comments
├── .env.local              ← API key pre-filled if you provided it
├── package.json
└── tsconfig.json
```

The generated `lib/nexa.ts` varies by auth provider:

```ts title="Clerk variant"
import { createNexa } from "@nexa-ed/next";
import { auth } from "@clerk/nextjs/server";

export const nexa = createNexa({
  apiKey: process.env.NEXA_API_KEY!,
  webhookSecret: process.env.NEXA_WEBHOOK_SECRET!,
  getUser: async () => {
    const { userId } = await auth();
    if (!userId) throw new Error("Unauthorized");
    return { userId };
  },
});
```

```ts title="NextAuth variant"
import { createNexa } from "@nexa-ed/next";
import { getServerSession } from "next-auth";
import { authOptions } from "@/lib/auth";

export const nexa = createNexa({
  apiKey: process.env.NEXA_API_KEY!,
  webhookSecret: process.env.NEXA_WEBHOOK_SECRET!,
  getUser: async () => {
    const session = await getServerSession(authOptions);
    if (!session?.user?.id) throw new Error("Unauthorized");
    return { userId: session.user.id };
  },
});
```

## After scaffolding

Follow these steps to get the app running:

**1. Install dependencies**

```bash
cd my-school && pnpm install
```

**2. Fill in environment variables**

Open `.env.local` and set:

```bash
NEXA_API_KEY=nxa_live_...
NEXA_WEBHOOK_SECRET=whsec_...
```

Get both values from **Nexa dashboard → Settings → API Keys** and **→ Webhooks**.

**3. Register webhook URLs**

In the Nexa dashboard, add:

| Event | URL |
|-------|-----|
| `file.completed` | `https://your-domain.com/api/nexa/webhook` |
| `payment.completed` | `https://your-domain.com/api/nexa/payments/forward` |
| `email.bulk_completed` | `https://your-domain.com/api/nexa/email-provisioning/forward` |

Use [ngrok](https://ngrok.com) to expose `localhost:3000` while developing.

**4. Start the dev server**

```bash
pnpm dev
```

Open [http://localhost:3000/dashboard](http://localhost:3000/dashboard) — your results viewer is live.
