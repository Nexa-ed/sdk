---
title: UI Components
description: Drop-in React components for teacher dashboards, file upload, and payment flows.
---

All components are pure Tailwind — no shadcn/ui dependency. Wrap your app in `<NexaProvider>` once and every component picks up the correct API base path automatically.

## Setup

```tsx title="app/layout.tsx"
import { NexaProvider } from "@nexa-ed/react";

export default function RootLayout({ children }) {
  return (
    <NexaProvider basePath="/api/nexa"> {/* basePath is optional, defaults to /api/nexa */}
      {children}
    </NexaProvider>
  );
}
```

## NexaPaymentWidget

Standalone payment widget: loads fee config, displays the total, opens Paystack checkout, and polls for completion.

```tsx
import { NexaPaymentWidget } from "@nexa-ed/react";

<NexaPaymentWidget
  email="student@example.com"
  netAmount={5000_00}          // kobo — school receives this amount
  metadata={{ userId, category: "enrollment" }}
  onSuccess={(ref) => router.push(`/success?ref=${ref}`)}
  onError={(err) => toast.error(err.message)}
/>
```

### Props

| Prop | Type | Description |
|------|------|-------------|
| `email` | `string` | Customer email for Paystack |
| `netAmount` | `number` | Amount school receives (kobo). Widget auto-calculates gross. |
| `metadata` | `PaymentMetadata` | Passed through to webhook |
| `onSuccess` | `(reference: string) => void` | Called when payment is confirmed |
| `onError` | `(error: PaymentError) => void` | Called on failure |
| `reference` | `string?` | Custom reference; auto-generated if omitted |

## EnrollmentPaymentFlow

3-step flow: fee breakdown → payment → confirmation receipt.

```tsx

<EnrollmentPaymentFlow
  email="student@example.com"
  studentName="Jane Doe"
  examFee={2000_00}             // kobo
  enrollmentFee={3000_00}       // kobo — optional
  metadata={{ userId, category: "enrollment" }}
  onComplete={(ref) => markEnrolled(ref)}
/>
```

Admin UI to view and update the platform fee configuration.
