---
title: UI Components
description: Drop-in React components for teacher dashboards, file upload, and payment flows.
---

All components are pure Tailwind — no shadcn/ui dependency. Wrap your app in `<NexaProvider>` once and every component picks up the correct API base path automatically.

## Setup

```tsx title="app/layout.tsx"
import { NexaProvider } from "@nexa-ed/react";

export default function RootLayout({ children }) {
  return (
    <NexaProvider basePath="/api/nexa"> {/* basePath is optional, defaults to /api/nexa */}
      {children}
    </NexaProvider>
  );
}
```

## NexaPaymentWidget

Standalone payment widget: loads fee config, displays the total, opens Paystack checkout, and polls for completion.

```tsx
import { NexaPaymentWidget } from "@nexa-ed/react";

<NexaPaymentWidget
  email="student@example.com"
  netAmount={5000_00}          // kobo — school receives this amount
  metadata={{ userId, category: "enrollment" }}
  onSuccess={(ref) => router.push(`/success?ref=${ref}`)}
  onError={(err) => toast.error(err.message)}
/>
```

### Props

| Prop | Type | Description |
|------|------|-------------|
| `email` | `string` | Customer email for Paystack |
| `netAmount` | `number` | Amount school receives (kobo). Widget auto-calculates gross. |
| `metadata` | `PaymentMetadata` | Passed through to webhook |
| `onSuccess` | `(reference: string) => void` | Called when payment is confirmed |
| `onError` | `(error: PaymentError) => void` | Called on failure |
| `reference` | `string?` | Custom reference; auto-generated if omitted |

## StudentEmailAccountManager

Create, list, suspend, restore, and reset passwords for student email accounts. All actions go through your `/api/student-emails/*` routes — no API key is exposed to the browser.

```tsx
import { StudentEmailAccountManager } from "@nexa-ed/react";

<StudentEmailAccountManager
  tier="tier-2-stalwart"
  domain="loretto.edu.ng"
  onAccountCreated={(account) => console.log("Created:", account.email)}
/>
```

### Props

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `tier` | `EmailTier` | Yes | Email tier: `"tier-1-nexa"` \| `"tier-2-stalwart"` \| `"tier-3-google"` |
| `domain` | `string` | Yes | Email domain, e.g. `"loretto.edu.ng"` |
| `basePath` | `string` | No | API proxy base path. Defaults to `""` |
| `onAccountCreated` | `(account: StudentEmail) => void` | No | Called after successful account creation |

<Callout type="warn">
  **Live key required.** Accounts created through this component call your live `/api/student-emails/create` endpoint.
  Use a test key during development — you will receive a sandboxed response with `_testMode: true`.
</Callout>

## EmailBulkProvisioningPanel

Provision up to 500 student accounts at once with live job progress tracking. Fires a bulk job and polls for completion every 3 seconds automatically.

```tsx
import { EmailBulkProvisioningPanel } from "@nexa-ed/react";

// Option A — paste JSON manually in the textarea
<EmailBulkProvisioningPanel
  tier="tier-2-stalwart"
  domain="loretto.edu.ng"
  onJobComplete={(progress) =>
    console.log(`Done: ${progress.successCount}/${progress.totalStudents}`)
  }
/>

// Option B — pre-load a students array (skips textarea)
<EmailBulkProvisioningPanel
  tier="tier-2-stalwart"
  domain="loretto.edu.ng"
  students={[
    { studentId: "stu_001", firstName: "Amara", lastName: "Obi" },
    { studentId: "stu_002", firstName: "Chidi", lastName: "Eze" },
  ]}
/>
```

### Props

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `tier` | `EmailTier` | Yes | Email tier |
| `domain` | `string` | Yes | Email domain |
| `students` | `EmailCreateFormData[]` | No | Pre-load students array — skips textarea |
| `basePath` | `string` | No | API proxy base path. Defaults to `""` |
| `onJobComplete` | `(progress: EmailJobProgress) => void` | No | Called when job reaches `completed` or `failed` |
